# Resume baseline

This directory holds the **canonical resume data** that the pipeline tailors per job application. The output is generated by [rendercv](https://github.com/rendercv/rendercv) — a Python tool that turns YAML into ATS-clean PDFs (Typst-backed in 2.8+, LaTeX-backed in 2.x).

The baseline lives at [carlos_jr_sysadmin.yaml](carlos_jr_sysadmin.yaml).

---

## Why YAML, why rendercv?

- **YAML is editable by hand** and by LLM. Structured enough to be machine-tweakable, readable enough to spot problems.
- **rendercv** produces single-column, no-icon, text-extractable PDFs that pass through ATS parsers cleanly. Canva and Word templates with multi-column layouts or icon fonts often break on resume parsers.
- **Themes used:** `engineeringresumes` (clean, single-column, IT-friendly). Other ATS-safe options: `classic`, `sb2nov`, `engineeringclassic`. Avoid `moderncv` — uses icons/colors that some ATS choke on.

## What's encoded in the baseline

The baseline matches the **Jr Sysadmin (~2 years technical experience)** framing we landed on:

- **Title line:** "Junior Systems Administrator" (not "Operations & Systems Lead")
- **Summary:** Leads with technical/sysadmin language, acknowledges ops + military background as transferable
- **BEAT THE BOMB:** Re-positioned as "Sysadmin / Technical Operations Manager", tech bullets first, ops bullets last
- **1-800-GOT-JUNK:** Demoted to 2 short bullets (vendor logistics + impact metric only)
- **US Army Reserve:** 2 short bullets — vet credibility + transferable training, no medical clinical detail
- **Skills:** Organized for sysadmin/IT eyes (OS → Networking → Hardware → Remote Admin → Scripting → AV → Business Systems → Documentation)
- **Projects:** AI job-pipeline project included (toggleable per JD)

**Honesty rules baked in:**
- 2 years of hands-on technical experience claimed (matches BEAT THE BOMB tenure)
- No claims to AD/GPO, hypervisor admin, cloud architecture, IaC, CI/CD, enterprise IAM
- Linux scoped to Photon OS / Ubuntu kiosks, not enterprise fleet
- Networking scoped to troubleshooting + maintenance, not architecture
- No fictional certifications (cert section commented out — fill in when earned)

## Section anatomy and tailoring flexibility

Each section is annotated for the **tailoring layer** (LLM-driven per-JD patch — see [TAILORING_SPEC.md](TAILORING_SPEC.md)):

### `summary`
- **Tailorable:** entire string can be swapped per JD
- LLM should produce a 3–5 line variant emphasizing the 2–3 most JD-relevant keywords

### `experience`
- **Tailorable:** bullet *order* and bullet *visibility*
- **Not tailorable:** company names, titles, dates — these are facts, never invent or shift
- BEAT THE BOMB bullets are **split into "tech-leaning" (top 6) and "ops-leaning" (bottom 2)**. The tailoring layer can drop the bottom 2 for purely IC sysadmin roles, or keep them for hybrid tech-ops postings.

### `education`
- **Not tailorable** — degree is a fact

### `technical_skills`
- **Tailorable:** row order and row visibility
- Each row is a `label: details` pair. LLM can reorder to surface JD-relevant rows first or drop a low-relevance row to save space.
- LLM must **not** invent new skill rows — only reorder/toggle what's there.

### `projects`
- **Tailorable:** show/hide the entire section per JD
- Show for tech-heavy interviews where "what have you built?" comes up
- Hide for purely conventional helpdesk JDs to save space

### `certifications` (commented out)
- Empty until Carlos earns CompTIA A+, Network+, Security+, Linux+, etc.
- LLM **must never** add certifications that aren't here

## Rendering the baseline

From the repo root:

```powershell
# One-time install
pip install rendercv

# Render the baseline
rendercv render resume_baseline/carlos_jr_sysadmin.yaml --output-folder-name resume_baseline/output
```

Output appears in `resume_baseline/output/`:
- `Carlos_Roman-Conville_CV.pdf` — final ATS-clean PDF
- `Carlos_Roman-Conville_CV.typ` (or `.tex` on rendercv 2.x older) — intermediate source
- `Carlos_Roman-Conville_CV.html` — web view (optional)
- `Carlos_Roman-Conville_CV.md` — markdown view (useful for ATS keyword review)

## Editing the baseline (when to actually change this file)

**Only update [carlos_jr_sysadmin.yaml](carlos_jr_sysadmin.yaml) when:**
- Carlos changes jobs
- Carlos earns a new certification
- A bullet is factually wrong
- A bullet is missing context that should be permanent (not JD-specific)
- The framing strategy changes (e.g. promote to "Regular Sysadmin" after more years)

**Never edit this file for a single application.** That's what the tailoring layer is for.
